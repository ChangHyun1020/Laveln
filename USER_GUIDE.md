# VesselV2 앱 사용 설명서 (USER GUIDE)

> 이 문서는 VesselV2 앱의 모든 기능을 제 3자도 쉽게 이해할 수 있도록
> 화면별로 상세히 설명합니다.

---

## 📱 앱 개요

**VesselV2**는 항만 라싱(Lashing) 업무를 위한 Android 앱입니다.

| 항목 | 내용 |
|------|------|
| 대상 사용자 | 항만 라싱 작업자, 현장 관리자 |
| 주요 기능 | DGT 선석 스케줄 확인, 모선 정보 관리, 근무 기록 관리 |
| 데이터 출처 | DGT(동부산컨테이너터미널) API + Firebase |
| 플랫폼 | Android (API 26 이상, Android 8.0+) |

## 🚀 1. 앱 실행 및 메인 화면

### 진입 경로
앱 아이콘 터치 → **MainActivity** (메인 화면)

### 화면 구성
```
┌─────────────────────────────────┐
│  ☰  VesselV2         [검색 아이콘] │ ← 툴바 (☰: 사이드바 열기)
├─────────────────────────────────┤
│                                 │
│   [선석 배정 현황 그래프]          │ ← BerthScheduleView
│   B1 ──────[OCEANBIRD]───────── │   (가로: 시간, 세로: 선석 B1~B5)
│   B2 ──────────────[STARLITE]── │
│   B3                            │
│   ...                           │
│                                 │
├─────────────────────────────────┤
│   [입항 예정 카드 리스트]          │ ← RecyclerView
│   OCEANBIRD   B1(S)   WORKING   │
│   ETB: 26/04/01 08:00           │
│   ...                           │
│                                 │
│                              ➕  │ ← FAB (모선 등록)
└─────────────────────────────────┘
```

### 주요 동작
| 동작 | 결과 |
|------|------|
| 앱 실행 시 자동 | DGT API에서 오늘~7일간 선석 스케줄 자동 로드 |
| 화면 당기기 (Swipe-Refresh) | 스케줄 데이터 수동 새로고침 |
| 그래프의 선박 바 터치 | QC 크레인별 작업 현황 다이얼로그 표시 |
| 카드 리스트 모선명 터치 | Firebase에서 해당 모선 정보 조회 → 상세 화면 이동 또는 등록 안내 |
| FAB(➕) 버튼 | 신규 모선 등록 화면으로 이동 |
| 뒤로가기 2번 | 앱 종료 (2초 이내 2번 눌러야 종료) |

## 📊 3. 선석 그래프 (BerthScheduleView)

### 그래프 구성
- **Y축(세로)**: 선석 번호 (B1~B5)
- **X축(가로)**: 시간 (6시간 간격, 06/12/18/00 표시)
- **색상 구분**:
  - 🔵 WORKING (작업 중)
  - 🟠 BERTHED (접안 완료)
  - 🟢 PLANNED (입항 예정)

### 선박 클릭 시 — QC 현황 다이얼로그

```
┌─────────────────────────────────┐
│  OCEANBIRD QC 현황               │
├─────────────────────────────────┤
│  QC01 (총 작업량 : 120)          │
│    완료 합계    │  잔여 합계      │
│      80        │    40           │
│  완료 양하 │ 적하 │ 잔여 양하 │ 적하│
│    50     │  30  │    25    │  15 │
├─────────────────────────────────┤
│  QC02 (총 작업량 : 95)           │
│    ...                          │
└─────────────────────────────────┘
```

> **QC(Quay Crane)**: 선박을 하역하는 크레인
> **양하(Discharge)**: 선박 → 부두 (내리기)
> **적하(Load)**: 부두 → 선박 (싣기)

## ✏️ 5. 모선 등록 화면 (AddVesselActivity)

### 진입 경로
- 메인 화면 FAB → AddVesselActivity
- 모선 목록 FAB → AddVesselActivity
- 리스트 카드 클릭 → 미등록 안내 → "등록" 선택 (모선명 자동 입력)

### 입력 항목
| 필드 | 설명 | 입력 방식 |
|------|------|---------|
| 모선명 | 선박 이름 (예: OCEANBIRD) | 직접 입력 |
| 입항 방문 날짜 | 입항 예정일 | DatePicker |
| 코너(corn) | 코너 캐스팅 타입 | 팝업 메뉴 선택 |
| 플랜지(flan) | 플랜지 타입 | 팝업 메뉴 선택 |
| 층수(floor) | 작업 층수 | 직접 입력 |
| 행(row) | 행 정보 | 직접 입력 |
| 트윈(twin) | 트윈 리프트 여부 | 팝업 메뉴 선택 |
| 턴버클 | 턴버클 정보 | 직접 입력 |
| 특이사항 | 메모 | 직접 입력 |
| 사진 | 현장 사진 (최대 4장) | 갤러리 선택 |

### 저장 흐름
1. [저장] 버튼 클릭
2. Firestore에 문서 저장 (`Constants.VESSEL_COLLECTION / {모선명 대문자}`)
3. 사진이 있으면 Storage에 압축 업로드 (`images/{모선명}/`)
4. 성공 시 이전 화면 복귀

## 🔐 보안 및 Git 업로드 가이드

### ✅ Git에 포함되지 않는 파일

| 파일 | 이유 |
|------|------|
| `google-services.json` | Firebase 프로젝트 비밀 키 포함 |
| `local.properties` | SDK 경로 등 로컬 환경 설정 |
| `app/src/main/res/drawable/erm.jpg` | 직원 비상 연락망 정보 |
| `*.keystore` | 앱 서명 키 |
| `.idea/` | IDE 로컬 설정 |

> 이 파일들은 `.gitignore`에 이미 포함되어 있습니다.
> **절대로 git add로 강제 추가하지 마세요.**

### ⚠️ 알려진 보안 이슈

**SSL 인증서 검증 우회 (DgtDataSource.kt)**
- 현상: DGT 내부망 서버가 자체 서명 인증서를 사용하여 Android SSL 오류 발생
- 조치: 모든 인증서를 신뢰하는 방식으로 우회 (내부망 전용이므로 위험도 제한적)
- 권고: DGT 서버 측에서 공인 CA 인증서 적용 시 해당 코드 즉시 제거 예정

## ❓ FAQ (자주 묻는 질문)

**Q: 앱 실행 시 스케줄이 로드되지 않아요**
A: 항만 내부 Wi-Fi(DGT 인트라넷)에 연결되어 있는지 확인하세요.
   DGT API는 외부 인터넷으로는 접근되지 않습니다.

**Q: 모선 등록 시 사진이 안 올라가요**
A: 인터넷 연결을 확인하세요. Firebase Storage 업로드는 외부 인터넷이 필요합니다.

**Q: 그래프를 눌러도 QC 현황이 안 나와요**
A: DGT 서버에서 해당 선박의 컨테이너 데이터가 아직 입력되지 않은 경우입니다.
   잠시 후 다시 시도하거나, DGT 웹사이트에서 직접 확인하세요.

**Q: 비상 연락망 메뉴 클릭 시 이미지가 보이지 않아요**
A: `erm.jpg` 파일은 보안 상 Git에 포함되지 않습니다.
   별도로 `app/src/main/res/drawable/erm.jpg` 경로에 파일을 배치해야 합니다.
